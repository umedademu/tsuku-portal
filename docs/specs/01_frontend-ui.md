# 01 フロント画面の考え方

## 1. 目的
- サイトを訪れた人が迷わず必要な情報にたどり着けるようにする。
- スマートフォンでもパソコンでも同じ内容を快適に見られるようにする。
- 後からページを追加・修正しやすい作りを目指す。

## 2. 主なページ
- トップページ：サービスの概要、導入実績、利用の流れ、問い合わせへの誘導。
- サービス紹介ページ：詳しい説明、料金の考え方、導入プロセス。
- 事例一覧と詳細：業種や課題別に探せるようにし、詳細ページでは成果や工夫点を紹介。
- 記事一覧と詳細：SEO向けの記事を分類し、詳細ページは目次や関連記事を表示。
- 問い合わせページ：チャット形式で入力を進められる窓口。
- 利用規約やプライバシーポリシーなどの固定ページ。

## 3. レイアウトとナビゲーション
- どのページでも同じヘッダーとフッターを使い、主要リンクと問い合わせボタンを常に表示する。
- スクロールしても位置がわかるように、必要に応じてヘッダーを固定表示する。
- 事例や記事の詳細ページでは、現在地が分かるパンくずリストを表示する。
- スマートフォンではハンバーガーメニューなど、片手でも操作しやすい導線を用意する。

## 4. コンポーネントのまとめ方
- ヒーローセクション、サービス紹介カード、事例カード、CTAバナー、問い合わせ導線など再利用する要素は部品として整理する。
- テキストや色を切り替えられるように設定を持たせ、デザイナーの変更依頼に柔軟に対応できるようにする。
- 表示や動きが複雑な部品は、使い方や制約を短くまとめたメモを準備する。

## 5. 画面サイズへの対応
- 画面幅に応じて、1カラム・2カラム・3カラムとレイアウトを柔軟に切り替える。
- モバイルではCTAボタンや問い合わせ導線を親指で押しやすい位置に置く。
- 画像やアイコンはぼやけないように高解像度でも表示できるファイルを用意する。

## 6. 動きと操作感
- フェードインやスライドなどの演出は最小限にし、読み込みを遅くしないよう注意する。
- キーボード操作でもフォーカス位置が分かるよう、フォーカスリングを表示する。
- 動きが多い要素でも、ユーザーが操作を続けられるように制御する。

## 7. アクセシビリティ
- 見出し階層やランドマーク要素を整え、読み上げソフトでも意味が伝わるようにする。
- 色のコントラストを十分に確保し、文字サイズも読みやすく調整する。
- 装飾目的の画像には空の代替テキストを設定し、意味のある画像には説明文を付ける。

## 8. フォームとCTA
- CTAは「主ボタン」「補助ボタン」「テキストリンク」など用途ごとに整理して使い分ける。
- チャット導線はページのスクロール状況や滞在時間を参考に表示タイミングを調整する。
- 入力フォームはエラーが発生した場所と理由をすぐに知らせる。

## 9. 表示速度と計測
- レイアウトがずれないように画像サイズや読み込み順を整える。
- フォントやサードパーティのスクリプトは、本当に必要なものだけを読み込む。
- Core Web Vitalsなどの指標を定期的に計測し、目標値を下回らないようにする。

## 10. 状態管理
- 全体で共有する情報（テーマ設定、同意状況、チャットの開閉など）は必要最小限にする。
- ページの読み込みや更新はNext.jsの仕組みを活用し、データの取り扱い方は機能ごとに決める。
- チャット入力など中断が予想される操作は、途中保存の仕組みを検討する。

## 11. 開発と保守
- UI部品を一覧化し、メンバー全員が同じ部品を再利用できる状態にする。
- 自動テストやLint、アクセシビリティチェックをCIに組み込み、結果を共有する。
- 主要ページのスクリーンショットや指標を定期的に確認し、品質を保つ。

## 12. 未決事項
- デザイン用のトークン（色や余白など）の管理方法をデザインチームと決める。
- 多言語対応の有無と開始時期をビジネス側と相談する。
- ダークモードを導入するかどうかをユーザー調査の結果を踏まえて判断する。
