# 01 現状静的LPの仕様メモ

## 1. 目的と前提
- `apps/web/src/app/page.tsx` に実装されている単一ページの内容を、そのまま要件として整理したメモ。
- コンテンツはすべて静的にハードコードされており、問い合わせフォームやAPI連携は存在しない状態。
- グローバルスタイルは `globals.css` にまとめられ、Tailwindは未使用（`@import "tailwindcss";` はあるがUtilityは使っていない）。

## 2. ページ全体構成
1. 固定ヘッダー（SNSリンク + ナビ + 「無料診断開始」ボタン）
2. ヒーローセクション（AI紹介文、2種類のCTA、残り受付数）
3. 「選ばれる理由」セクション（3枚の特徴カード）
4. 「監修・運営責任者」セクション（プレースホルダー画像と代表コメント）
5. 「施工事例」セクション（3件の事例カード）
6. 「地域別サービス案内」セクション（地域キーワードのタグ一覧）
7. フッター（会社情報、サイトマップ、関連情報）

## 3. 各セクションの要件
- **ヘッダー**
  - SNSリンク：Instagram / TikTok / LINE への外部リンクをFont Awesomeアイコン付きで表示。
  - ナビ：`#why-ai` `#works` `#expert-profile` のページ内リンク + 右端に「無料診断開始」ボタン。
  - 上部固定、グラデーション背景、余白内に`social-links`と`nav-menu`を水平配置。
- **ヒーロー**
  - バッジ文言：「Google最新AI『Gemini 2.5 Pro』搭載診断システム」。
  - 見出し：「その見積もり内容、本当に適正ですか？」の途中でハイライト。
  - 説明文：見積もり・図面の診断を無料で行う旨を2行構成で表示。
  - CTAボタン2種：事業者向け（B2B）と個人向け（B2C）、`data-funnel`属性付き、Font Awesomeアイコンを付与。
  - 緊急性バー：`stats.remainingToday`（初期値5）を黄色ハイライトで表示。
- **選ばれる理由（why-ai）**
  - 説明文に当月利用者数（`stats.monthlyUsers`、初期値128名）をハイライト表示。
  - 3種類の特徴カード：コスト可視化 / リスク可視化 / 即時専門知識活用。タイトル内の改行は`formatLineBreaks`で制御。
- **専門家紹介（expert-profile）**
  - 代表者のプレースホルダー写真（外部URL）。
  - 会社名：`companyName = 株式会社相模建設ツクルンジャー` を肩書き下に表示。
   - コメント文：「17年以上の経験…」の固定文章と保有資格／建設業許可番号（プレースホルダー）を記載。
- **施工事例（works）**
  - 3件の事例情報をカード表示。各カードは地域・年月・タイトル・要約・成果指標（削減率/電気代予測/是正費回避）を縦並び。
  - データは`caseStudies`配列で固定値を管理。
- **地域別サービス案内**
  - `regionKeywords`配列に含まれる8件の地域別フレーズをタグ状に並べ、SEO向けキーワードクラウドとして扱う。
- **フッター**
  - 左：会社名・住所・電話番号（ダミー値）とSNSリンク再掲。
  - 中央：サイトマップ（HOME含む4リンク）。
  - 右：固定テキスト「プライバシーポリシー」「XMLサイトマップ」。
  - 下部にコピーライト（当年 + 会社名）を表示。

## 4. スタイルと動作
- 主要カラーはカスタムプロパティで定義（濃紺 #1a1a2e、青 #0f3460、アクセント金 #ffd700 等）。
- レイアウトはFlex/Gridを活用し、セクションごとに`.container`で最大幅1200pxに中央寄せ。
- メディアクエリは1つ（768px以下でナビ・CTAレイアウト調整）。
- すべて静的要素のため、ボタンを押してもチャットやフォームは起動しない。

## 5. データと更新方法
- コンテンツは`page.tsx`内の定数配列（`navLinks`, `socialLinks`, `caseStudies`, `regionKeywords` など）で定義。
- 受付残数や月間利用者数も定数（`stats`）で管理し、自動更新ロジックは存在しない。
- 画像はNext.jsの`Image`コンポーネントで一箇所のみ使用（プロフィール写真）。

## 6. 既知の不足/今後の課題
- docsで定義されているチャット型問い合わせ・AI補助・API連携は未着手。
- CTAボタンに紐づく動作（モーダル起動、問い合わせ導線切り替えなど）が無いため、ユーザーは次のステップへ進めない。
- SEO向け記事・事例詳細・問い合わせ記録などの下層ページは存在せず、ナビ内リンクはすべて同一ページ内スクロールのみ。
